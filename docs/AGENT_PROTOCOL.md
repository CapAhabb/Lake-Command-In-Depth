# Agent Protocol

## Mission

Agents support the captain. They do not steer product direction.

## Cost Rule

Use free/local work first:

- local code edits
- tests
- static analysis
- small documentation updates
- deterministic scripts

Avoid paid or long-running agents until there is a working model and revenue
path.

## Commit Rule

No agent may commit, push, merge, publish, or open a PR without captain approval.

## Scope Rule

Each agent gets one small job:

- exact files or folders owned
- expected output
- stop condition
- verification command

Agents must not rewrite unrelated files.

## Product Rule

The current visual target is the dark chartplotter bezel:

- dark plastic housing
- tight left data panel
- white/beige layer buttons
- orange glow when active
- large chart screen
- toolbox and compass as supporting controls

## Handoff Rule

Agent output must include:

- files changed
- tests run
- risks or unknowns
- next recommended action
